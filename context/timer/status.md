# timer 구현 추적

## 범례

- ✅ 반영됨 — 코드에 구현 완료
- ⬜ 미반영 — 정책/설계만 확정, 코드 미구현

## 모드

| ID | 항목 | 상태 | 비고 |
|----|------|------|------|
| FR-1 | 평상시 모드(Idle): 투두 관리 + 소음 모니터링만, 키보드/마우스 추적 OFF | ⬜ | |
| FR-2 | 평상시 80dB 초과 시 소음 경고 멘트 노출 | ⬜ | character의 `noiseLoud` 버킷 |
| FR-3 | 평상시 우측 상단 상태 텍스트 (7개 중 무작위 1개, 이모티콘 없음) | ⬜ | 팝업 내부 chip |
| FR-4 | 집중 모드: 사용자 설정 집중+휴식 = 1 세션, 하이브리드 측정 ON | ⬜ | |
| FR-5 | 집중 모드 우측 상단 상태 텍스트 = "집중 중" 고정 | ⬜ | |
| FR-21 | 뽀모도로 진행 중에는 OS 알림 보내지 않음 | ⬜ | phase 가드 |

## 시간 설정

| ID | 항목 | 상태 | 비고 |
|----|------|------|------|
| FR-22 | 집중 시간/휴식 시간 사용자 직접 설정 | ⬜ | 5~90분 / 3~30분 |

## 디자인 결정

| ID | 항목 | 상태 | 비고 |
|----|------|------|------|
| DEC-2 | 일시정지 버튼 제거 — 일시정지 상태 자체를 두지 않음 | ⬜ | 시안의 paused 분기 코드 미반영 |
| DEC-10 | 슬립/깨어남 처리 — wall-clock 차이가 grace(180초) 초과 시 자동 discard | ⬜ | macOS NSWorkspace / Windows WM_POWERBROADCAST |
| DEC-17 | FR-3 우상단 텍스트 = 팝업 내부 우상단 chip 위치 | ⬜ | 명세 표현 명확화 |
